Why is the JavaScript textContent property safer than innerHTML?

TextContent only changes the text, whereas innerHTML inserts HTML content. InnerHTML is less secure because a bad actor has more power to insert their own malicious code.


Explain the relationship between an HTML attribute and a corresponding DOM property (explain the connection to JavaScript). Use a specific example in your answer

If you want to access an individual HTML element and change it's text content, you would first access the element by naming its id with the getElementById method. If you tag this method after the "document" keyword it means that you are getting the element from the related HTML file:

document.getElementById("elementId")

then you would assign it a new string with the textContent property:

.textContent = "New text.";

The whole line of code would look like this:

document.getElementById("elementId").textContent = "New text.";
